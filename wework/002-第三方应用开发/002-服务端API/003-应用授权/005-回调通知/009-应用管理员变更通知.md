当授权方（即授权企业）在企业微信管理端修改了应用负责人，通过该接口通知服务商。服务商接收到变更通知之后，需自行调用[获取应用的管理员列表](#14945/获取应用的管理员列表)进行授权内容变更比对。
> 应用为管理员授权模式时：包括超级管理组变更、分级管理组修改应用权限以及直接修改应用负责人
应用为成员授权模式时：包括修改分级管理员管理的应用（成员授权暂时不支持设置应用负责人）

**请求方式：**POST（**HTTPS**）
**请求地址： **https://127.0.0.1/suite/receive?msg_signature=3a7b08bb8e6dbce3c9671d6fdb69d15066227608&timestamp=1403610513&nonce=380320359 （数据回调url）

**请求包体：**

```
<xml>
  <ToUserName><
- ![CDATA[toUser]]></ToUserName>
  <FromUserName><
- ![CDATA[sys]]></FromUserName>
  <CreateTime>1348831860</CreateTime>
  <MsgType><
- ![CDATA[event]]></MsgType>
  <Event><
- ![CDATA[change_app_admin]]></Event>
  <AgentID>1</AgentID>
</xml>
```

参数说明：

 | 参数 | 说明 |
 |---- | ----- |
 | ToUserName | 企业微信CorpID |
 | FromUserName | 此处固定为sys|
 | CreateTime | 消息创建时间（整型）|
 | MsgType | 消息类型，此时固定为：event |
 | Event | 事件类型，此处固定为change_app_admin|
 | AgentID | 企业应用的id，整型。可在应用的设置页面查看 |
