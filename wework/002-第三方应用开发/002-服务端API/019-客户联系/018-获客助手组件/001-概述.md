# 了解「获客助手组件」

申请接入获客助手组件后，企业可授权在你的场景中接入获客助手。授权后，该场景的用户可通过获客链接添加企业成员，你可自动获得数据回传。

## 接入流程
### 1、 提交申请

注册成为企业微信服务商，在「服务商助手-工具-通用配置-获客助手组件」处提交申请。
### 2、 引导企业授权接入

申请通过并发布组件后，你可获得专属的组件授权链接/二维码。可在你的业务页面配置该链接，企业点击后将自动打开企业微信，授权在你的场景中接入获客助手，你可获得改场景中的获客转化数据。
### 3、 配置获客链接

企业授权给你的链接会自动带有你的场景参数，配置时请保留该参数。你可接受在你的场景中的获客转化数据。
### 4、 接收数据回传

可在获客链接的comp_scene=xxx后再次拼参，企业微信将以此参数进行获客转化数据回调，用于跨平台用户身份识别。如：

::: details 
https://work.weixin.qq.com/ca/caxxxxxxxxxxxx?comp_scene=xxx&customer_channel=1
:::

字符串不超过64字节，超过会被截断，如第三方平台用户ID超过64字节，建议做内部映射。

## 授权流程
开发者可以在服务商助手的「工具-通用配置-获客助手组件」获取授权链接或二维码，之后企业可使用该授权链接或二维码接入获客助手组件，具体流程如下图所示：
- ![获客助手链接授权.png](https://wework.qpic.cn/wwpic/535021_3UCLy-BqRfu-Q6H_1700208185/0)
**注：**
- 回调临时授权码的公共信息字段详细说明见[授权成功通知](#14951)。
- 利用临时授权码获取永久授权码见[获取永久授权码](#14942)；
- 临时授权码10分钟后会失效，第三方服务商需尽快使用临时授权码换取永久授权码及授权信息。
每个企业授权的每个应用的永久授权码、授权信息都是唯一的，第三方服务商需妥善保管。后续可以[通过永久授权码获取企业access_token](#14944)，进而调用企业微信相关API为授权企业提供服务。
- 企业授权成功以后，若再次通过授权链接或二维码可进入获客助手授权链接管理界面，变更授权的获客链接时会回调[变更授权通知](#14951/变更授权通知)。
- 企业取消接入获客助手组件时，会回调[取消授权通知事件](#14951/#取消授权通知)。注意，服务商收到取消授权事件后，应当确保删除该企业所有相关的数据。

## 更多说明
###1、关于获客助手组件授权链接/二维码
开发者可以在上文[授权流程](#授权流程)中所述的授权链接拼上指定参数，实现对应功能。
- 拼接参数state，当企业授权接入获客助手组件时，[授权成功通知或变更授权通知](#14951)将回调该参数给开发者，开发者可依据该参数，明确知道企业是要在哪里接入获客链接。
例: https://open.work.weixin.qq.com/compApp/wwxxxxxxxx?state=xxxxxx

###2、关于获客链接
a）可将获客链接配置在微信内/外的获客场景，客户点击即可打开成员的profile。
b）企业授权接入后，服务商可通过API或在服务商助手的「工具-通用配置-组件数据」处查看企业接入的获客链接，并请及时将获取到的获客链接配置到自己的场景中。
c）获客链接带有服务商的场景参数，配置时须保留该参数。通过该获客链接发起的操作，你才可进行管理。
d）若要接收获客转化回调，则必须在链接后添加customer_channel=自定义字符串。企业微信将以此参数进行获客转化数据回调，用于跨平台用户身份识别。
