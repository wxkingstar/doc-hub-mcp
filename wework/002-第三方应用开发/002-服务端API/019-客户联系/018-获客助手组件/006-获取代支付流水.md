服务商可通过此接口获取使用量代付流水。企业通过代付产生使用量后，次日可在获客助手中查看代付的订单记录，若需要与企业侧使用量统计周期一致，请按当天的0时0分01秒到第二天的0时0分0秒的时间戳查询。

**请求方式：**POST(**HTTPS**)
**请求地址：**https://qyapi.weixin.qq.com/cgi-bin/service/customer_acquisition/get_bill_list?suite_access_token=SUITE_ACCESS_TOKEN

**请求示例：**
```
{
 "begin_time":1605171726,
 "end_time":1605172726,
 "auth_corpid": "xxxxxxxxx",
 "cursor":"CURSOR",
 "limit":10
}
```

**参数说明：**

| 参数 | 必须 | 说明 |
| :----------- | :--- | :----------------- |
|suite_access_token|是|[获客助手组件的应用凭证](#14939) |
| begin_time | 是 | 流水记录开始时间 |
| end_time | 是 | 流水记录结束时间 |
|auth_corpid|是|授权企业corpid|
|cursor|否|用于分页查询的游标，字符串类型，由上一次调用返回，首次调用可不填|
|limit|否|返回的最大记录数，默认值100，最大不超过1000|

**补充说明:**
>流水的起止间隔不能超过31天。

**权限说明：**
- 获客助手组件可调用

**返回结果：**
```
{
 "errcode":0,
 "errmsg":"ok",
 "next_cursor":"CURSOR",
 "bill_list":[
 {
 "timestamp": 1605171726,
 "link_id": "xxxxxxx",
 "state": "REMARK",
     "price":30000
 }
 ]
}
```

**参数说明：**

| 参数 | 说明 |
| :-|
| errcode | 返回码 |
| errmsg | 对返回码的文本描述内容 |
| next_cursor | 分页游标，在下次请求时填写以获取之后分页的记录，如果已经没有更多的数据则不返回该字段|
|bill_list|代支付流水记录列表|
|bill_list[].timestamp |消耗时间|
|bill_list[].link_id |获客链接id，获客链接已删除的情况不返回此字段|
|bill_list[].state |加好友时的state参数，为空则不返回|
|bill_list[].price |服务商代付金额，单位为分|
