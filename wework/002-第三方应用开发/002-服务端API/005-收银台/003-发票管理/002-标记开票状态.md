服务商可以使用该接口标记某个应用订单发票的开票状态

__请求方式__:POST(__HTTPS__)

__请求地址:__ https://qyapi.weixin.qq.com/cgi-bin/paytool/mark_invoice_status?provider_access_token=PROVIDER_ACCESS_TOKEN

__请求示例__

```json
{
  "order_id": "ORDERID",
  "oper_userid": "USERID",
  "invoice_status": 1,
  "invoice_note": "NOTE"
}
```

__参数说明__:

|参数|必须|类型|说明|
|:-----------|:---|:-------|:------------|
| provider_access_token | 是 | string | [服务商调用接口凭证](#15143) |
|order_id|是|string|要标记开票状态的订单号|
| oper_userid | 是 | string | 标记开票状态的操作人，操作人需要有「收银台-发票管理」的权限 |
| invoice_status | 是 |uint32|要标记的开票状态。<br>1-已开具纸质发票，并邮寄给客户<br>2-已开具电子发票，并发送至客户邮箱<br>3-取消开具发票，取消后企业可再次申请<br>若订单对应开票状态为已开票，此次标记将不生效|
| invoice_note | 是 | string| 填写开票备注，例如发票邮寄单号等，客户侧可见。不超过200字节|

__权限说明__:
- 服务商需有在收银台完成商户号注册

__返回结果__:

```json
{
 "errcode": 0,
 "errmsg": "ok"
}
```

__参数说明__:

|参数|类型|说明|
|:-----------|:-------------|:----------|
|errcode|int32|返回码|
|errmsg|string|错误码描述|
