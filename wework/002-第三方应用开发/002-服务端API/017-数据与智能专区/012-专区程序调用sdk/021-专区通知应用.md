通过调用该sdk接口，专区程序可将一些事件通知应用后台。

**请求方法**
通过SDK调用，具体方式参考[专区程序使用指引](#54006)。

**请求接口名：**
spec_notify_app

**请求包体：**
 ```
{
  "notify_id": "NOTIFY_ID",
  "notify_scene": 1
}
```
**参数说明：**

| 参数 | 必须 | 说明 |
| ------------ | ---- | --------------------------------------------- |
|notify_id| 否 | 可指定`notify_id`，见[SDK生成notify_id](#notify_id_note)。如果不指定，则由接口自动生成。不超过64字节。 |
|notify_scene| 否 | 通知场景值，可用于标识此次通知对应的事件类型。取值范围：0~255，满足条件则通过[应用接收专区通知](#53360)回调给开发者，否则忽略该字段。|

**返回结果：**
```
{
 "errcode": 0,
 "errmsg": "ok",
 "notify_id": "NOTIFY_ID"
}
```
**参数说明：**

| 参数 | 说明 |
| ------- | ---- |
| errcode |错误码 |
|errmsg|错误码说明|
|notify_id|10分钟内有效，同时会通过[应用接收专区通知](#53360)回调给开发者。如果请求包体指定了`notify_id`，则直接采用并返回，否则由接口自动生成。|

## SDK生成notify_id

### Python
``` python
from wwspecapisdk import spec_gen_notify_id

notify_id = spec_gen_notify_id()
```
### C++
``` c++
#include "cpp_sdk.h"

std::string notify_id = GenerateNotifyId();
```

### Java
``` java
import com.tencent.wework.SpecUtil;

String notify_id = SpecUtil.GenerateNotifyId();
```
