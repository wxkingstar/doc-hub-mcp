对于API创建的网络研讨会，成员角色变更时，触发该事件。

**示例**
xml请求示例：
```
<xml>
 <ToUserName><
- ![CDATA[toUser]]></ToUserName>
 <FromUserName><
- ![CDATA[userId]]></FromUserName> 
 <FromUserTmpOpenId><
- ![CDATA[tmpOpenId]]></FromUserTmpOpenId> 
 <OperatedUser>
   <UserId><
- ![CDATA[userId]]></UserId> 
   <TmpOpenId><
- ![CDATA[tmpOpenId]]></TmpOpenId> 
 <UserRole>2</UserRole>
 </OperatedUser> 
 <CreateTime>1348831860</CreateTime>
 <MsgType><
- ![CDATA[event]]></MsgType>
 <Event><
- ![CDATA[meeting_change]]></Event>
 <ChangeType><
- ![CDATA[webinar_role_change]]></ChangeType>
 <MeetingId><
- ![CDATA[wcjgewCwAAqeJcPI1d8Pwbjt7nttzAAA]]></MeetingId>
</xml>
```

**说明**

 | 参数 | 说明 |
 |---- | ----- |
 | ToUserName | 企业微信CorpID |
 | FromUserName | 操作者UserID。如果为系统触发事件，默认为```sys```；如果为成员操作事件，仅当是企业成员时有值。|
 | FromUserTmpOpenId | 操作者TmpOpenID。操作者的会中临时ID，仅成员操作事件有值。|
 | OperatedUser | 被操作者|
 | OperatedUser.UserId | 被操作者UserID。仅当是企业成员时有值。|
 | OperatedUser.TmpOpenId | 被操作者TmpOpenID。被操作者的会中临时ID。|
 | OperatedUser.UserRole | 被操作者变更后的角色</br>0: 普通成员角色</br>1: 是创建者权限 </br>2: 主持人权限 </br>3: 创建者权限+主持人权限 </br>4: 游客 </br>5: 游客+主持人权限 </br>6: 联席主持人 </br>7: 创建者+联席主持人 </br>8: restApi接口指派的主持人 </br>30：研讨会内部嘉宾 </br>31: 研讨会外部嘉宾 </br>32：研讨会邀请链接入会嘉宾 </br>33：研讨会观众 </br>34：有音视频权限的研讨会观众|
 | CreateTime | 消息创建时间，unix时间戳|
 | MsgType | 消息类型，此时固定为：```event``` |
 | Event | 事件类型，此时固定为：```meeting_change``` |
 | ChangeType | 事件类型，此时固定为：```webinar_role_change``` |
 | MeetingId | 会议ID |
