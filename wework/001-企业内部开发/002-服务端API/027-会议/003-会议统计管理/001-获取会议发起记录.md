该接口用于获取员工发起的会议记录及其状态，包含员工主动发起快速会议及作为首位参与者进入预约会议的场景。

**请求方式：** POST（**HTTPS**）
**请求地址：** https://qyapi.weixin.qq.com/cgi-bin/meeting/statistics/get_start_list?access_token=ACCESS_TOKEN

**请求包体：**

```json
{
 "type": 1,
 "begin_time": 1700000000,
 "end_time": 1700001000,
 "limit": 1000,
 "cursor": "CURSOR"
}
```

**参数说明：**

| 参数 | 必须 | 类型|说明 |
|:--|
| access_token | 是 | string|调用接口凭证。获取方法查看“[获取access_token](#15074)” |
|type|是|int|查询类型。</br>1:发起成功的会议记录</br>2:发起失败的会议（企业同时发起的会议数已达上限，员工无法发起）|
|begin_time|是|int|查询范围起始时间戳，单位为秒|
|end_time|是|int|查询范围结束时间戳，单位为秒|
|limit|否|int|每次拉取的数据量，默认值200，最大值1000|
|cursor |否 |string|用于分页查询的游标，由上一次调用返回，首次调用可不填 |

**补充说明:**
>查询时间跨度不能超过30天，查询区间是左闭右开的
>记录会按时间从大到小排序
>会过滤会议发起者不在应用可见范围中的记录，因此返回的记录数可能会小于limit设置的最大记录数。

**权限说明：**
- 会议发起者需要在应用可见范围内
- 自建应用需要配置在“可调用接口的应用”列表

**返回结果：**

```
{
  "errcode": 0,
  "errmsg": "ok",
  "next_cursor": "abcd",
  "meeting_list": [
    {
      "userid": "USERID1",
      "start_time": 1700000000
    },
    {
      "userid": "USERID2",
      "start_time": 1700000000
    }
  ]
}
```

**参数说明：**

|参数|类型|说明|
|:--|
|errcode|int|返回码|
|errmsg|string|对返回码的文本描述内容 |
|next_cursor |string |当前数据最后一个key值，如果下次调用带上该值则从该key值往后拉，用于实现分页拉取 |
|has_more|bool|是否还有数据待拉取|
|meeting_list|object[]|发起会议成功或失败的记录信息，取决于请求中的type参数|
|meeting_list.userid|string|会议发起者的userid。如果是企业外部成员作为首位参与者进入预约会议，则该字段为此预约会议创建者的userid|
|meeting_list.start_time|int|会议发起时间|
