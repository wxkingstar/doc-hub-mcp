在指定网络研讨会（Webinar）中导入报名信息。
- 会议未开启报名时会返回未开启报名错误。

**请求方式：** POST（**HTTPS**）
**请求地址：** https://qyapi.weixin.qq.com/cgi-bin/meeting/webinar/enroll/import?access_token=ACCESS_TOKEN

**请求包体：**
```json
{
  "meetingid": "meetingid11234",
  "enroll_list": [{
    "userid": "USERID",
    "area": "86",
    "phone_number": "13333333333",
    "nick_name": "NICKNAME"
  }]
}
```

**参数说明：**

| 参数| 必须 | 类型|说明|
|:--|
| access_token | 是 | string|调用接口凭证。获取方法查看“[获取access_token](#15074)”|
| meetingid | 是 | string|网络研讨会ID|
|enroll_list|是|object[]|报名成员列表。详见EnrollRequest。|

**EnrollRequest说明**
|参数|必须|类型|说明|
|:--|:--|
|userid|否|string|报名成员 userid|
|area|否|string|国家/地区代码。若使用 phone_number 的方式导入，该字段必填（例如：中国传86，不是+86）。|
|phone_number|否|string|手机号。|
|nick_name|否|string|报名的昵称。此处的修改不会影响会中展示的昵称，仅展示在导出的报名信息。|

**权限说明**
- 仅配置在“可调用接口的应用”列表中的自建应用可调用
- 仅允许修改该应用创建的网络研讨会的数据

**返回结果：**
```
{
  "errcode": 0,
  "errmsg": "ok",
  "total_count": 1,
  "enroll_list": [{
    "enroll_id": "123456",
    "userid": "USERID",
    "area": "86",
    "phone_number": "13333333333",
    "nick_name": "NICKNAME",
    "enroll_code": "CODE"
  }]
}
```

**参数说明**

| 参数|类型| 说明|
|:--|
| errcode |int32| 返回码|
| errmsg |string| 对返回码的文本描述内容|
|total_count|uint32|成功导入的报名信息条数|
|enroll_list|object[]|报名成员列表。详见EnrollResponse。|

**EnrollResponse说明**
|参数|类型|说明|
|:--|
|enroll_id|string|报名ID|
|userid|string|报名成员 userid|
|area|string|国家/地区代码。若使用 phone_number 的方式导入，该字段必填（例如：中国传86，不是+86）。|
|phone_number|string|手机号。|
|nick_name|string|报名的昵称。|
|enroll_code|string|报名码，用于电话入会的凭证。|
