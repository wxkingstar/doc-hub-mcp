将会议中的高级自定义布局应用到指定成员或者整个会议。
也可以恢复指定成员或整个会议的默认布局。

>高级布局应用到指定成员目前仅支持 H.323/SIP 会议室终端。

**请求方式：** POST（**HTTPS**）
**请求地址：** https://qyapi.weixin.qq.com/cgi-bin/meeting/advanced_layout/apply?access_token=ACCESS_TOKEN
 
**请求包体：**
```
{
  "meetingid": "MEETINGID",
  "layout_id": "LAYOUTID1",
  "user_list": [{
    "tmp_openid": "TMPOPENID1"
  }]
}
```

**参数说明：**
|参数 |必须 |类型|说明 |
|:--|
|access_token |是 |string|调用接口凭证。获取方法查看“[获取access_token](https://work.weixin.qq.com/api/doc/90001/90143/90605)” |
|meetingid|是|string|会议 ID |
|layout_id| 是|string|选择应用的布局 ID（若传空""，表示恢复成当前会议的默认布局）。<br>应用布局的优先级从高到低为：个性布局 > 自定义布局 > 默认布局<br>（MRA 不支持同框模式， 如果会议设置为同框模式，MRA 应用默认布局。）|
|user_list|否|object[]|用户列表对象数组。如果该字段为空，为会议设置高级自定义布局；如果该字段携带用户，则只为指定用户设置个性布局。单次最多支持20个用户。详见**User**|

**User 说明**
|参数 |必须|类型|说明 |
|:--|
|tmp_openid |是 |string|用户当前会议中的临时身份 ID，单场会议唯一。仅对H.323/SIP 会议室终端生效|
**返回结果：**
```
{
  "errcode": 0,
  "errmsg": "ok"
}
```

**参数说明：**
|参数 |类型|说明 |
|:--|
|errcode|int32 |返回码 |
|errmsg |string|对返回码的文本描述内容 |
