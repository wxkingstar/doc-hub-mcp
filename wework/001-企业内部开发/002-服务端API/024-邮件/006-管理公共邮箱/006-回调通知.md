当公共邮箱发生事件后，企业微信会将相应的事件推送给应用配置的```回调URL```。

接收并解析事件的方法见：[接收事件](#12977)

## 公共邮箱接收邮件事件
当公共邮箱收到邮件后，触发该事件。

**示例**
xml请求示例：
```
<xml>
 <ToUserName><
- ![CDATA[toUser]]></ToUserName>
 <FromUserName><
- ![CDATA[sys]]></FromUserName> 
 <CreateTime>1668831860</CreateTime>
 <MsgType><
- ![CDATA[event]]></MsgType>
 <Event><
- ![CDATA[public_email_change]]></Event>
 <ChangeType><
- ![CDATA[receive_email]]></ChangeType>
 <Id>1</Id>
 <Amount>2</Amount>
</xml>
```

**说明**

 | 参数 | 说明 |
 |---- | ----- |
 | ToUserName | 企业微信CorpID |
 | FromUserName | 此事件该值固定为sys，表示该消息由系统生成|
 | CreateTime | 消息创建时间，unix时间戳|
 | MsgType | 消息类型，此时固定为：```event``` |
 | Event | 事件类型，此时固定为：```public_email_change``` |
 | ChangeType | 事件类型，此时固定为：```receive_email``` |
 | Id | 公共邮箱id |
 | Amount | 公共邮箱当前的新邮件数 |
