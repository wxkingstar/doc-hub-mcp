该接口可以查询通过[智能表格自动化创建的群聊](https://open.work.weixin.qq.com/help2/pc/19684?person_id=1&searchData=#4%E3%80%81%E8%87%AA%E5%8A%A8%E5%8C%96%E6%B5%81%E7%A8%8B%E5%88%9B%E5%BB%BA%E7%BE%A4%E8%81%8A) 列表。

**请求方式：** POST（**HTTPS**）
**请求地址：** https://qyapi.weixin.qq.com/cgi-bin/wedoc/smartsheet/groupchat/list?access_token=ACCESS_TOKEN

**请求包体**

```json
{
  "docid":"DOCID",
  "cursor":"CURSOR",
  "limit":2
}
```

**参数说明：**

| 参数 | 必须 | 说明 |
| ------------ | ---- |------------------------------------------------------------------------------------------------------------------|
| access_token | 是 | 调用接口凭证|
|docid|是|智能表格ID。需为当前应用创建的智能表格|
|cursor|否|用于分页查询的游标，字符串类型，由上一次调用返回，首次调用可不填|
|limit|否|用于分页查询，每次请求返回的数据上限。默认100，最大200<br>!!#ff0000 注意：不保证每次返回的数据刚好为指定limit!!，必须用返回的`has_more`判断是否继续请求|

**权限说明**

- 仅自建应用可调用，需配置到文档“[可调用应用](#43883)”列表中的应用
- 可见范围需包含根部门

**并发限制：20** 

**返回结果：**

```
{
 "errcode": 0,
 "errmsg": "ok",
 "has_more":true,
 "next_cursor":"GNIJIGEO",
 "chat_id_list":["chatid1","chatid2"]
}
```
**参数说明：**

| 参数 | 说明 |
| ------- | ---------------------- |
| errcode | 返回码 |
| errmsg | 对返回码的文本描述内容 |
|has_more|是否还有更多数据未获取|
|next_cursor|下一次请求的cursor值|
|chat_id_list|符合条件的群聊chatid列表|
