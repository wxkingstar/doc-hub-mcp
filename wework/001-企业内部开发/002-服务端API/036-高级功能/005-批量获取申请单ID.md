自建应用可使用此接口，批量获取企业成员高级功能申请单的id信息。

**请求方式：**POST（**HTTPS**）
**请求地址：** https://qyapi.weixin.qq.com/cgi-bin/advanced_feature/get_apply_id_list?access_token=ACCESS_TOKEN 

**请求示例：**

``` json
{
 "business_type":1,
 "limit":10,
 "cursor":"CURSOR",
 "req_type":1
}
```

**参数说明：**
|参数 |类型 |是否必须 |说明 |
|:-|
|business_type |uint32 |是 |申请的高级账号类型<br>1-邮件 2-文档 3-微盘 4-会议 |
|userid |string |是 |申请的userid |
|limit |uint32 |否 |分页查询的数据上限。默认100，最大200<br>注意：不保证每次返回的数据刚好为指定limit，须用返回的has_more判断是否继续请求 |
|cursor |string |否 |用于分页查询的游标，字符串类型，由上一次调用返回，首次调用可不填 |
|req_type |uint32 |否 |0-所有 1-仅api单 2-非api 申请单， 默认为0 |

**权限说明**:
调用的应用需要满足如下的权限：

|应用类型 |权限要求 |
|:-|
|自建应用 |配置到「!!#ff0000 高级功能-成员申请管理-通过api设置申请流程!!」中 |
|代开发应用 |暂不支持|
|第三方应用 |暂不支持|
 
**返回示例：**

```json
{
 "errcode": 0,
 "errmsg": "ok",
 "next_cursor": "CURSOR",
 "apply_id_list":[
 "APPLY_ID1",
 "APPLY_ID2"
 ],
 "has_more": true
}
```
**参数说明：**

|参数 |说明 |
|:-|
|errcode |错误码 |
|errmsg |错误码说明 |
|next_cursor |分页请求的下一页游标 |
|apply_id_list |申请id列表 |
|has_more |是否还有下一页数据 |
