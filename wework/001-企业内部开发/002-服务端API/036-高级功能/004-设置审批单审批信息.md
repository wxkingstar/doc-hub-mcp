自建应用可使用此接口，设置企业成员高级功能申请单的审批信息。

**请求方式：**POST（**HTTPS**）
**请求地址：** https://qyapi.weixin.qq.com/cgi-bin/advanced_feature/set_approval_detail?access_token=ACCESS_TOKEN 

**请求示例：**

``` json
{
 "approval_id": "APPROVALID",
 "approval_status":1,
 "apply_id":"ApplyId",
 "approval_url":"www.qq.com",
 "process_list":{
 "node_list":[
 {
 "current_approvers":["LEADER1", "LEADER2"],
 "completed_approvers":["LEADER0"],
 "node_apv_status":1,
 "node_apv_rel":1,
 "apv_update_time":1715741759
 },
 {
 "current_approvers":["LEADER3", "LEADER4"],
 "completed_approvers":["LEADER5"],
 "node_status":1,
 "node_apv_rel":1,
 "apv_update_time":1715741759
 }
 ]
 }
}
```

**参数说明：**
|参数 |类型 |是否必须 |说明 |
|:-|
|apply_id |string |是 |申请id |
|approval_id |string |是 |审批id，注意：应用生成审批id后，审批id和申请id是一一对应的，不可改变 |
|approval_status |uint32 |是 |审批单状态：1-审批中; 2-已驳回; 3-已同意; 101-已撤销 |
|approval_url |string |是 |审批单跳转链接，须已"http://"或"https://"开头 |
|process_list.node_list |array(obj) |是 |审批单审批节点，注意：如果需要变更审批节点信息，需要全量节点都传入 |
|process_list.node_list.node_apv_status |uint32 |是 |审批节点状态：1-审批中; 2-已驳回; 3-已同意; 101-已撤销; 102-未到流程 |
|process_list.node_list.node_apv_rel |uint32 |是 |审批节点多人审批方式：1-会签；2-或签；3-依次审批 |
|process_list.node_list.current_approvers |array(string ) |否 |当前审批节点待处理人列表，最多100个，待处理人列表和已处理人列表不可同时为空 |
|process_list.node_list.completed_approvers |array(string) |否 |当前审批节点已处理人列表，做多100个，待处理人列表和已处理人列表不可同时为空 |
|process_list.node_list.apv_update_time |uint32 |否 |审批节点更新时间 |

**权限说明**:
调用的应用需要满足如下的权限：

|应用类型 |权限要求 |
|:-|
|自建应用 |配置到「!!#ff0000 高级功能-成员申请管理-通过api设置申请流程!!」中 |
|代开发应用 |暂不支持|
|第三方应用 |暂不支持|
 
::: warning 注意
注：仅当企业自建应用通过“设置审批单审批信息”接口成功返回信息时，API审批流程才算做设置成功。若企业自建应用在收到“成员申请的提交回调”后的24小时内，未通过“设置审批单审批信息”接口成功返回信息，则API流程设置失败，成员申请将恢复为设置接口之前的申请流程。
:::

**返回示例：**

```json
{
 "errcode": 0,
 "errmsg": "ok"
}
```
**参数说明：**

|参数 |说明 |
|:-|
|errcode |错误码 |
|errmsg |错误码说明 |
