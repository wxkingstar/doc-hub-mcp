>企业可以通过这个接口快速递交材料，用于创建对外收款账户申请单
>图片ID通过[提交图片](#47982/提交图片)获取，只能使用该应用本身提交的图片
>同一个提现人员最多申请200个商户号，同一个企业最多申请2000个商户号

**请求方式：**POST(**HTTPS**)
**请求地址：**https://qyapi.weixin.qq.com/cgi-bin/miniapppay/apply_mch?access_token=ACCESS_TOKEN

**权限说明：**
调用的应用需要满足如下的权限

| 应用类型 | 权限要求 |
| -------- | ---- | 
|自建应用|配置到「[对外收款- 可调用接口的应用](#24952/配置可调用接口的应用)」中 |
|代开发应用|暂不支持| 
|第三方应用|暂不支持|

 *注：* 从2023年12月1日0点起，不再支持通过系统应用secret调用接口，存量企业暂不受影响 [查看详情](https://developer.work.weixin.qq.com/document/51165)

**参数说明：**

|参数|必须|说明|
|---|---|---|
|access_token|是|调用接口凭证 |
|out_request_no|是|业务申请编号，长度限制为1~32个字符，可以通过[查询申请单状态](#47984/查询申请单状态)查询|
|organization_type|是|主体类型，参考[主题类型对照表](#organization_type)|
|business_license_info|是|营业执照/登记证书信息，<br>1、主体为“个体/企业”时，请上传营业执照。<br>2、主体为“事业单位/社会团体组织”时，请上传登记证书。<br>参考[营业执照/登记证书参数说明](#business_license_info)|
|finance_institution_info|否|金融机构许可证信息，<br>当主体是金融机构时必填，<br>参考[金融机构许可证参数说明](#finance_institution_info)|
|merchant_short_name|是|商户简称，<br>UTF-8格式，中文占3个字节，即最多21个汉字长度。将在支付完成页向买家展示，需与商家的实际售卖商品相符 。|
|id_card_info|是|经营者/法人证件信息，参考[经营者/法人证件信息参数说明](#id_card_info)|
|owner|否|经营者/法人是否为受益人，<br>主体类型为企业时，需要填写：<br>1、若经营者/法人是最终受益人，则填写：true。<br>2、若经营者/法人不是最终受益人，则填写：false。|
|ubo_info|否|受益人证件信息，<br>仅当主体类型为企业，且经营者/法人不为受益人时需要填写，填写规范与经营者/法人证件信息相同，可参考[经营者/法人证件信息参数说明](#id_card_info)<br>根据国家相关法律法规，需要提供公司受益所有人信息，受益所有人需符合至少以下条件之一：<br>1、直接或者间接拥有超过25%公司股权或者表决权的自然人。<br>2、通过人事、财务等其他方式对公司进行控制的自然人。<br>3、公司的高级管理人员，包括公司的经理、副经理、财务负责人、上市公司董事会秘书和公司章程规定的其他人员。|
|contact_info|是|超级管理员信息，超级管理员需在开户后进行签约，并可接收日常重要管理信息和进行资金操作，请确定其为商户法定代表人或负责人。<br>参考[超级管理员信息参数说明](#contact_info)|
|account_info|是|结算账户信息，请填写商家提现收款的银行账户信息。<br>参考[结算账户信息参数说明](#account_info)|
|sales_scene_info|是|经营场景证明，参考[经营场景证明参数说明](#sales_scene_info)|
|business_id|是|经营范围，参考[经营范围对照表](#business_id)<br>请注意，不同的主体类型可以选择的经营范围是不同的|
|qualifications|否|特殊资质，<br>1、根据商户经营业务要求提供相关资质，详情查看[《行业对应特殊资质》](https://kf.qq.com/faq/220228qEfuAz220228bMFji6.html)。<br>2、请提供为“申请商家主体”所属的特殊资质，可授权使用总公司/分公司的特殊资 质；<br>3、最多可上传5张照片，请填写通过[提交图片](#47982/提交图片)预先上传图片生成好的图片ID 。|
|business_addition_pics|否|补充材料，<br>根据实际审核情况，额外要求提供。最多可上传5张照片，请填写通过[提交图片](#47982/提交图片)预先上传图片生成好的图片ID 。|
|userid|是|提现人员，<br>必须指定一位提现人员，填写成员UserID，对应管理端的账号。<br>该提现人员必须实名认证，且实名认证信息必须与超级管理员为同一人。<br>该提现人员必须在应用的可见范围内。|
