>通过这个接口查询已提交的申请单状态，只能使用该应用本身提交的申请单号
>使用[提交创建对外收款账户的申请单](#47983/提交创建对外收款账户的申请单)接口中填写的out_request_no来查询

**请求方式：**POST(**HTTPS**)
**请求地址：**https://qyapi.weixin.qq.com/cgi-bin/miniapppay/get_applyment_status?access_token=ACCESS_TOKEN

**权限说明：**
调用的应用需要满足如下的权限，且只允许获取由应用本身提交的申请单

| 应用类型 | 权限要求 |
| -------- | ---- | 
|自建应用|配置到「[对外收款- 可调用接口的应用](#24952/配置可调用接口的应用)」中 |
|代开发应用|暂不支持| 
|第三方应用|暂不支持|

 *注：* 从2023年12月1日0点起，不再支持通过系统应用secret调用接口，存量企业暂不受影响 [查看详情](https://developer.work.weixin.qq.com/document/51165)

**请求包体：**

```
{
  "out_request_no":"xxxxx"
}
```

**参数说明：**

|参数|必须|说明|
|---|---|---|
|access_token|是|调用接口凭证 |
|out_request_no|是|业务申请编号，长度限制为1~32个字符，在[提交创建对外收款账户的申请单](#47983/提交创建对外收款账户的申请单)中提交|

**参数说明：**
|参数|说明|
|---|---|
|errcode|返回码|
|errmsg|对返回码的文本描述|
|status|申请单的具体状态，参考[申请单状态参数说明](#status)|
|apply_state|申请单当前所处阶段，对应关系参考[申请阶段对应关系](#apply_state)|
|real_sign_state|当前签约阶段，对应关系参考[签约阶段对应关系](#real_sign_state)|
|reject_reason|驳回理由|

**申请单状态参数说明：**
|参数|说明|
|---|---|
|applyment_state|<details><summary>申请状态</summary>枚举值参考[申请状态枚举值](#applyment_state)</details>|
|applyment_state_desc|申请状态描述|
|sign_state|<details><summary>签约状态</summary>枚举值参考[签约状态枚举值](#wx_sign_state)</details>|
|sign_url|<details><summary>签约链接</summary>1、当申请状态为NEED_SIGN 或 签约状态为UNSIGNED时返回，该链接为永久有效；<br>2、申请单中的超级管理者，需用已实名认证的微信扫码打开，完成签约。</details>|
|sub_mchid|<details><summary>电商平台二级商户号</summary>当申请状态为NEED_SIGN或FINISH时才返回。</details>|
|audit_detail|<details><summary>驳回原因详情</summary>各项资料的审核情况。当申请状态为REJECTED或 FROZEN时才返回。</details>|
|audit_detail.param_name|<details><summary>参数名称</summary>提交申请单的资料项名称。</details>|
|audit_detail.reject_reason|<details><summary>驳回原因</summary>提交资料项被驳回原因。</details>|
|account_validation|<details><summary>汇款账户验证信息</summary>当签约阶段字段（real_sign_state字段，注意不是status字段下的sign_state字段）的值为2（即已签约）时才有可能返回。<br>当申请状态为ACCOUNT_NEED_VERIFY 时有返回，可根据指引汇款，完成账户验证。<br>参考[汇款账户验证信息参数说明](#account_validation)</details>|
|legal_validation_url|<details><summary>法人验证链接</summary>当签约阶段字段（real_sign_state字段，注意不是status字段下的sign_state字段）的值为2（即已签约）时才有可能返回。<br>1、当申请状态为ACCOUNT_NEED_VERIFY，且通过系统校验的申请单，将返回链接。<br>2、建议将链接转为二维码展示，让商户法人用微信扫码打开，完成账户验证。<br>注：商户申请单进入审核状态后，微信侧会校验法人证件号码是否跟营业执照匹配，若匹配，返回该字段 ; 若不匹配，不支持法人扫码验证，不返回该字段。</details>|

**申请状态枚举值：**
|参数|对应状态|参数|对应状态|
|---|---|---|---|
|CHECKING|资料校验中|NEED_SIGN|待签约|
|ACCOUNT_NEED_VERIFY|待账户验证|FINISH|完成|
|AUDITING|审核中|FROZEN|已冻结|
|REJECTED|已驳回|CANCELED|已作废|

**签约状态枚举值：**
|参数|对应状态|
|---|---|
|UNSIGNED|未签约。该状态下，电商平台可查询获取签约链接，引导二级商户的超级管理员完成签约|
|SIGNED|已签约。指二级商户的超级管理员已完成签约。注意：若申请单被驳回，商户修改了商户主体名称、法人名称、超级管理员信息、主体类型等信息，则需重新签约|
|NOT_SIGNABLE|不可签约。该状态下，暂不支持超级管理员签约。一般为申请单处于已驳回、已冻结、机器校验中状态，无法签约|

**汇款账户验证信息参数说明：**
|参数|说明|
|---|---|
|account_name|<details><summary>付款户名</summary>需商户使用该户名的账户进行汇款。</details>|
|account_no|<details><summary>付款卡号</summary>结算账户为对私时会返回，商户需使用该付款卡号进行汇款。</details>|
|pay_amount|<details><summary>汇款金额</summary>需要汇款的金额(单位：分)。</details>|
|destination_account_number|<details><summary>收款卡号</summary>收款账户的卡号</details>|
|destination_account_name|<details><summary>收款户名</summary>destination_account_name</details>|
|destination_account_bank|<details><summary>开户银行</summary>收款账户的开户银行名称。</details>|
|city|<details><summary>省市信息</summary>收款账户的省市。</details>|
|remark|<details><summary>备注信息</summary>商户汇款时，需要填写的备注信息。</details>|
|deadline|<details><summary>汇款截止时间</summary>请在此时间前完成汇款。</details>|

**申请阶段对应关系：**
|参数|对应关系|参数|对应关系|
|---|---|---|---|
|0|初始状态|4|绑定失败|
|1|申请中|5|未申请|
|2|绑定成功|6|待法人验证|
|3|已撤销申请|||

**签约阶段对应关系：**
|参数|对应关系|
|---|---|
|0|不可签约|
|1|未签约|
|2|已签约|
