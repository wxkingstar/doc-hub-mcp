不同应用类型接收回调事件权限如下：

| 应用类型 | 权限要求 | 接收事件开关|
| -------- | ---- | 
|自建应用|配置到「[会议室- 可调用接口的应用](#22789)」中 |无|
|代开发应用|暂不支持| 无|
|第三方应用|暂不支持|无|

## 会议室预定事件
当用户在企业微信预定会议室时，会触发该事件回调给会议室系统应用。

请求示例：
```
<xml>
  <ToUserName><
- ![CDATA[toUser]]></ToUserName>
  <FromUserName><
- ![CDATA[fromUser]]></FromUserName>
  <CreateTime>1348831860</CreateTime>
  <MsgType><
- ![CDATA[event]]></MsgType>
  <Event><
- ![CDATA[book_meeting_room]]></Event>
  <MeetingRoomId>1</MeetingRoomId>
  <BookingId><
- ![CDATA[bkebsada6e027c123cbafAAA ]]></BookingId>
</xml> 
```
参数说明：

 | 参数 | 说明 |
 |----|---- |
 | ToUserName | 企业微信CorpID |
 | FromUserName | 成员UserID|
 |CreateTime|操作时间|
 |MsgType|消息的类型，此时固定为event |
 |Event|事件的类型，此时固定为book_meeting_room|
 |MeetingRoomId|会议室id |
 |BookingId|预定id，可[根据该ID查询具体的会议预定情况](#22517/根据会议ID查询会议室的预定信息)|
 
## 会议室取消事件
当用户在企业微信取消预定会议室时，会触发该事件回调给会议室系统应用；如果该会议室由自建应用预定，除了会议室系统应用外，也会回调给对应的自建应用。
注意：通过接口预定会议室后，用户在终端修改对应日程/会议时间时，会自动取消并重新预定会议室，并同时触发会议室取消事件和会议室预定事件的回调。

请求示例：

```
<xml>
  <ToUserName><
- ![CDATA[toUser]]></ToUserName>
  <FromUserName><
- ![CDATA[fromUser]]></FromUserName>
  <CreateTime>1348831860</CreateTime>
  <MsgType><
- ![CDATA[event]]></MsgType>
 <Event><
- ![CDATA[cancel_meeting_room]]></Event>
  <BookingId><
- ![CDATA[bkebsada6e027c123cbafAAA ]]></BookingId>
  <MeetingRoomId>1</MeetingRoomId>
</xml>
```
参数说明：

 | 参数 | 说明 |
 |----|---- |
 | ToUserName | 企业微信CorpID |
 | FromUserName | 成员UserID|
 |CreateTime|操作时间|
 |MsgType|消息的类型，此时固定为event |
 |Event|事件的类型，此时固定为cancel_meeting_room|
 |MeetingRoomId|会议室id |
 |BookingId|预定的id|
