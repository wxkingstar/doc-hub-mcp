此接口用于查看企业内成员在指定时间内日程闲忙状态

**调用前提：**
> 必须先调用过wx.qy.login，且session_key未过期，开发者可调用checkSession 检查当前登录态。
>当前成员必须在应用的可见范围之中，否则调用时会报“no permission”错误。
>应用需具有日程使用权限，否则调用时会报“no permission”错误。

### wx.qy.checkSchedule(Object object)
#### 参数

##### Object object

| 参数名 | 类型 | 必填 | 说明 | 
| -------- | -------- | ---- | -------------------------------- | 
| start_time | Uint32 | 是 | 开始时间，Unix时间戳 | 
| end_time | Uint32 | 是 | 结束时间，Unix时间戳 | 
| users | Array | 否 | 成员userid列表，第三方应用与代开发应用须传入open_userid列表。不填或列表为空则仅查看用户本人的日程闲忙状态 | 
| success | Function | 否 | 接口调用成功的回调函数 | 
| fail | Function | 否 | 接口调用失败的回调函数 | 
| complete | Function | 否 | 接口调用结束的回调函数（调用成功、失败都会执行） | 

#### 示例代码：

```js
wx.qy.checkSchedule({
    start_time: 1667232000,
    end_time: 1667318400,
    users: ["jack", "jason"],
    success: function(res) {
 // 回调
 }
});

```
