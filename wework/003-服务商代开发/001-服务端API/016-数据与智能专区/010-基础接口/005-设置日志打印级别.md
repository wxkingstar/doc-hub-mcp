专区里的程序可以调用SDK打印三种级别的日志：ERR、INFO、DBG，级别顺序为：ERR<INFO<DBG。
而一般程序稳定后，不需要再关心DBG甚至INFO日志，为了节省日志存储空间，开发者可以调用接口设置指定程序的日志级别。设置之后，专区程序只会存储不高于该级别的日志。

## 设置日志打印级别
设置指定模型或程序的日志打印级别。

__请求方式__:POST(__HTTPS__)

__请求地址:__ https://qyapi.weixin.qq.com/cgi-bin/chatdata/set_log_level?access_token=ACCESS_TOKEN

__请求示例__

```json
{
 "program_id": "xxxx",
 "log_level": 2
}
```

__参数说明__:

|参数|必须|类型|说明|
|:-----------|:---|:-------|:------------|
| access_token | 是 | string | 调用接口凭证 |
|program_id|是|string|应用关联的程序id|
|log_level|是|uint32|日志级别</br>指定后，仅会存储不高于改级别的日志。例如指定级别为2，那么只会存储级别为1或2的日志</br>默认级别为2</br>取值范围：</br>&#160;&#160;&#160;&#160;1 - ERR</br>&#160;&#160;&#160;&#160;2 - INFO</br>&#160;&#160;&#160;&#160;3 - DBG|

__权限说明__: 
- 应用需具有数据专区权限
- 指定的程序program_id需与应用有授权关系
- 官方模型不支持

__返回结果__:

```json
{
 "errcode": 0,
 "errmsg": "ok"
}
```

__参数说明__:

|参数|类型|说明|
|:-----------|:-------------|:----------|
|errcode|int32|返回码|
|errmsg|string|错误码描述|

## 获取当前日志打印级别
获取指定模型或程序的日志打印级别。

__请求方式__:POST(__HTTPS__)

__请求地址:__ https://qyapi.weixin.qq.com/cgi-bin/chatdata/get_log_level?access_token=ACCESS_TOKEN

__请求示例__

```json
{
 "program_id": "xxxx"
}
```

__参数说明__:

|参数|必须|类型|说明|
|:-----------|:---|:-------|:------------|
| access_token | 是 | string | 调用接口凭证 |
|program_id|是|string|应用关联的程序id|

__权限说明__: 
- 应用需具有数据专区权限
- 指定的程序program_id需与应用有授权关系
- 官方模型不支持

__返回结果__:

```json
{
 "errcode": 0,
 "errmsg": "ok",
 "log_level": 3
}
```

__参数说明__:

|参数|类型|说明|
|:-----------|:-------------|:----------|
|errcode|int32|返回码|
|errmsg|string|错误码描述|
|log_level|uint32|日志级别|
