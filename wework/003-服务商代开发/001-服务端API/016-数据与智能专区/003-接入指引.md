## 一、如何接入 

### 1. 上传模型/程序
在数据与智能专区中上传服务商的模型/程序，**程序为必须上传**，模型可使用企业微信提供的或上传服务商自己的。上传路径为：「桌面端服务商助手 - 工具 - 数据与智能专区」或「服务商后台 - 应用管理 - 数据专区 - 数据与智能专区」。

服务商在专区中部署模型/程序的机器资源由企业微信提供，机器资源当前为限时免费。服务商使用企业会话数据时，需要付费购买，详见[购买流程](#53518/%E4%B8%89%E3%80%81%E5%A6%82%E4%BD%95%E8%B4%AD%E4%B9%B0)。

::: tip 提示
「数据与智能专区」能力当前灰度开放，若服务商有匹配的应用场景，且希望加入灰度范围，可在页面底部扫码添加企业微信产品客服进行申请。
:::

### 2. 开启应用权限
在第三方应用或代开发应用的权限详情页中，勾选开启“数据与智能专区权限”。勾选后，设置需要分析的企业数据、使用的模型、使用的程序，以及是否使用会话展示组件。
- ![enter image description here](https://wework.qpic.cn/wwpic3az/844772_BAJhoO6wTJGqO9i_1717401330/0)

### 3. 应用安装及权限授权
企业安装服务商的第三方应用或代开发应用后，可授权服务商「数据与智能专区权限」。应用安装时默认不授权「数据与智能专区权限」，企业需要在安装后进行二次授权，可通过以下方式进行二次授权：
- 调用JSAPI授权：服务商应用可调用权限授权JSAPI，打开权限授权页引导企业完成授权 [查看接口](https://developer.work.weixin.qq.com/document/path/98603)
- 从权限详情页授权：企业可直接在该应用的权限详情页进行授权。授权路径为：「移动端企业微信 - 工作台 - 管理企业 - 应用管理 - 应用详情页 - 应用权限」 或 「企业管理后台 - 应用管理 - 应用详情页 - 授权信息」。
- ![enter image description here](https://wework.qpic.cn/wwpic3az/976417_LixcX5sMSXuvW65_1717402534/0)

### 4. 企业开启会话内容
若授权时企业还未开启「企业会话内容」，授权页将展示开启指引，服务商也可引导企业在「企业管理后台 - 安全与管理 - 管理工具 - 数据与智能专区 - 企业会话内容」进行开启。
- ![enter image description here](https://wework.qpic.cn/wwpic3az/52719_A0FXGVoTRbyFYIe_1717402930/0)

### 5. 设置公钥
企业授权之后，服务商需要调用[设置公钥](#51799)接口，为该企业设置加密消息的公钥，强烈建议每个企业设置不同的公钥。另外，也同时为该企业[设置专区接收回调事件](#53411)，以及时为该企业接收通知事件。

##二、如何使用

### 1. 准备工作
开发者需要准备一个程序，该程序首先需要引用专区SDK，实现获取会话记录等SDK接口的调用。
如果需要在程序中实时接收企业微信专区后台的回调通知，例如“产生会话回调通知”，则需要在程序中起动一个http服务并监听8080端口，用于接收专区后台的通知，接收到请求后，通过SDK对请求进行解密。 具体参考：[专区程序接入指引](#54006)、[专区程序示例](#54166)

### 3.专区程序使用专区SDK
#### 3.1 专区程序通过sdk调用企微后台
企业微信在数据与智能专区中提供一套SDK接口，应用可通过SDK接口获取会话记录与知识集。

| SDK接口 | 接口描述| 使用场景举例（供参考） |
| ------ | ------ | ------- | 
|[获取会话记录](#53341)|可获取授权范围内成员的会话记录，包括会话ID、消息ID、消息发送人、消息接收人、消息发送时间、消息类型|● 结合展示组件，展示成员会话内容<br>● 统计分析成员会话数据|
|[获取内部群消息](#53342)|可获取企业内部群信息，包括群主、群成员、群创建时间、群成员入群时间|● 结合获取会话记录接口、会话展示组件，按群的维度展示群中会话内容和群信息|
|[会话名称搜索](#53294)|基于名称搜索群|● 用于搜索群，查看指定群名的群会话内容|
|[会话消息搜索](#53335) |基于文本内容搜索会话消息 |● 用于搜索消息，查看指定内容对应的会话消息 |
|[关键词规则管理](#53346) |可通过定义关键词、关键行为来定义一个关键词规则，命中规则的消息将回调通知到专区程序 |● 敏感会话内容、敏感行为预警相关功能<br>● 基于特定内容、行为实现的产品功能 |
|[获取会话同意情况](#53343)|获取成员客户的会话存档同意情况 |● 基于成员客户会话存档同意情况实现的展示分析功能 |

#### 3.2 专区程序通过sdk调用专区模型/程序
企业微信提供内部大模型供应用进行会话内容的分析工作，另外也允许应用根据需要自行上传自有模型和自有程序到专区中并使用。

| SDK接口 | 接口描述| 使用场景举例（供参考） |
| ------ | ------ | ------- | 
|[情感分析模型](#53348) |获取会话消息的情感状态（正中负） |● 评估企业客户服务质量 |
|[会话摘要模型](#53349) |获取会话消息的摘要内容 |● 总结客户交流的目的 |
|[客户标签模型](#53350) |根据客户会话给客户打标签 |● 对聊天中的客户进行标签分类 |
|[话术推荐模型](#53351) |根据企业知识集推荐回复话术 |● 利用知识集即时回复客户 |
|[企微通用模型](#53707) |自定义提示词进行会话分析<br>(仅自建应用可用) |● 更有导向性的会话摘要分析等 |
|[自有模型分析](#53352) |自有模型和协议进行会话分析 |● 专有数据训练的自有模型 |
|[自有程序分析](#53659) |自有程序和协议进行会话分析 |● 定制的会话内容分析程序 |

### 4. 专区程序接收企业微信回调
若用户在企业微信发送消息或者其他操作触发相关事件时，企业微信后台会将用户产生的事件回调到专区程序。若专区程序需要接收事件，需要[设置专区接收回调事件](#53411)。设置应用接收回调的程序后，企业微信会将会话范围内用户相关事件回调到专区该程序所在的服务。参考：[专区程序接收事件通知](#53419)

### 5. 专区程序回调通知应用后台
若专区程序调用[专区通知应用](#53408)sdk ，则企业微信后台会向当前应用数据回调地址推送[应用接收专区通知](#53360)。调用该sdk后，企微信后台同时会返回notify_id给到专区程序，专区程序可使用notify_id在本地保存数据。应用后台收到通知后可拿到notify_id，并使用notify_id再去调用专区程序，专区程序可使用notify_id返回之前调用sdk时保存在本地的数据。

## 三、如何购买
企业授权服务商应用「数据与智能专区权限」30天内，服务商应用可免费使用企业会话数据进行分析和展示；免费使用时间到期后，服务商需购买企业会话内容数据接口，详细如下：
- **购买路径：**在「桌面端服务商助手 - 工具 - 通用配置 - 购买高级接口」或「服务商后台 - 应用管理 - 购买高级接口」购买数据与智能专区会话内容数据调用
- ![enter image description here](https://wework.qpic.cn/wwpic3az/354696_llL-EIubTf-jFvY_1717405872/0)
- **定价方案：**按照会话内容数据的账号及时长收费，一个账号可用于分析和展示一个员工的会话数据；不同类型的会话内容数据定价不同，详细如下：
| 类型 | 内容 | 定价 |
| ------ | ------ | ------ |
| 内外部会话及语音通话 | 员工之间的单聊、群聊及语音通话；员工与客户的单聊、群聊及语音通话 | 300元/账号/年 |
| 内外部会话 | 员工之间的单聊、群聊；员工与客户的单聊、群聊 | 150元/账号/年 |

服务商在接入过程中如有疑问，可扫码添加企业微信产品客服进行沟通。
