企业成员使用企业微信app扫描[打印机二维码](#15948)或者[设备登录二维码](#16638)时，此时企业微信云端向设备推送该事件

**指令包体：**

```
{
  "cmd": "push_user_scan",
  "headers": {
     "req_id": "xxxx"
   },
  "body":
  {
    "userid": "aa",
    "op_type": "op_type1",
    "oper_id": "xxx",
    "state":"STATE",
    "user_type":1
  }
}
```

**参数说明：**

| 参数名 | 类型 | 描述 |
| --------- | ------ | -------------------------------- |
| req_id | String | 透传请求的req_id |
| cmd | String | 指令号，此处固定为push_user_scan |
| userid | String | 员工id |
| op_type | String | 本次扫码的操作指令类型 scan_qrcode_open : 门禁扫码开门 |
| oper_id | String | 本次扫码的操作唯一id |
| state | String | 二维码上的自定义参数，参见“[生成打印机二维码](#15948)”或者“[设备登录二维码](#16638)” |
| user_type | uint32 | 扫码成员的类型。0：普通成员， 1：管理员 |
