设备处于待激活状态时调用该接口，当企业已经授权绑定设备后，可以获取到secret，用于设备登录，此参数应缓存至flash。
当企业取消授权后secret失效，当调用登录或者激活接口返回企业未授权时，设备应清空所有数据，重新进行激活。
当设备没有secret时，应每隔5s轮询，直到设备绑定企业产生secret为止。
5s轮询期间接收到[push_bind_corp事件](#15376)的话可以立即发起active操作，优化响应速度。

**请求包体：**

```
{
 "cmd":"active",
 "headers":
  {
    "req_id":"xxxxx"
  },
  "body":
  {
    "active_code":"xxxx"
  }
}
```

**参数说明：**
 
| 参数名 | 是否必须 | 类型 | 描述 |
| ----------- | ---- | ------ | ---------------- |
| req_id | 是 | String | 请求的id，自行保证不会重复即可 |
| active_code | 是 | String | 激活码 |

**返回结果：** 

```
{
  "headers": {
     "req_id" : "xxxx"
   },
  "body":
  {
    "secret":"xxxxxx"
  }
   "errcode": 0,
   "errmsg": "ok"
}
```

**参数说明：** 

| 参数名 | 描述 |
| ------- | ----------- |
| req_id | 透传请求的req_id |
| errcode | 错误码 |
| errmsg | 错误码描述 |
| secret | 凭证密钥 |
