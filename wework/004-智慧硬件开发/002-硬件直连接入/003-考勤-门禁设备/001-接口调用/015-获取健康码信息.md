当设备可通过此接口获取用户的健康码信息，包括健康状态以及健康码内容。开发者需要根据健康码内容自行生成二维码。

**请求包体：**

```
{
 "cmd":"get_health_status",
 "headers":
 {
 "req_id":"xxxxx"
 },
 "body":
 {
 "userid": "zhangsan",
    "user_type":2
 }
}

```

**参数说明：**
 
| 参数名 | 是否必须 | 类型 | 描述 |
| ------ | ---- | ------ | ---------------- |
| req_id | 是 | string | 请求的id，自行保证不会重复即可 |
| userid | 是 |string| 员工id |
| user_type |是| uint32 |用户类型， 0：员工（即老师） 2：学生 |

**返回结果：** 

```
{
  "headers": 
  {
   "req_id" : "xxxx"
  },
  "body":
  {
    "status":1,
    "qrcode_data":"QRCODE_DATA"
  }，
  "errcode": 0,
  "errmsg": "ok"
}
```

**参数说明：** 

| 参数名 | 描述 |
| ------------ | ----------- |
| req_id | 透传请求的req_id |
| body.status | 健康状态 0：正常，1：异常，2：未知/未填写|
| body.qrcode_data|健康码内容，开发者需要根据二维码内容，自行生成二维码图片|
| errcode | 错误码 |
| errmsg | 错误码描述 |
