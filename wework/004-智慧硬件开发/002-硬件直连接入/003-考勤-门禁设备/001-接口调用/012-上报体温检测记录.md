当设备识别到人员体温异常时，上报到企业微信，调用后复学码相关状态会更新。

**请求包体：**

```
{
 "cmd":"report_user_temperature",
 "headers":
 {
 "req_id":"xxxxx"
 },
 "body":
 {
 "userid": "zhangsan",
    "user_type":2,
    "temperature":"36.7",
    "status":0
 }
}

```

**参数说明：**
 
| 参数名 | 是否必须 | 类型 | 描述 |
| ------ | ---- | ------ | ---------------- |
| req_id | 是 | string | 请求的id，自行保证不会重复即可 |
| userid | 是 |string| 员工id |
| usertype |是| uint32 |用户类型， 0：员工（即老师） 2：学生 |
| temperature | 是| string | 温度，单位摄氏度
|status|是|uint32|状态，0：正常， 1：发热， 2：温度过低 |

**返回结果：** 

```
{
  "headers": 
  {
   "req_id" : "xxxx"
  },
  "body":
  {
    "status":0
  },
  "errcode": 0,
  "errmsg": "ok"
}
```

**参数说明：** 

| 参数名 | 描述 |
| ------------ | ----------- |
| req_id | 透传请求的req_id |
| errcode | 错误码 |
| errmsg | 错误码描述 |
| body.status | 提交后健康码状态 0：正常，1：异常，2：未知/未填写|
