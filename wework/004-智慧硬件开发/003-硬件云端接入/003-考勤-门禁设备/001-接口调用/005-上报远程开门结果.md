当用户从企业微信端发起远程开门时，企业微信服务端触发一条回调指令[远程开门指令](#40338)。当开门指令完成后，厂商云端通过此接口，将录入结果上报至企业微信服务端。
 
 
**请求方式：**POST（**HTTPS**）
**请求地址：** https://qyapi.weixin.qq.com/cgi-bin/openhw/device/report_remote_open_result?device_access_token=DEVICE_ACCESS_TOKEN
**请求包体：**

```
{
  "oper_id":"OPER_ID",
  "type":"remote_open_door",
  "errcode":1,
  "errmsg":"设备离线"
}
```

**参数说明：**
 
| 参数名 | 是否必须 | 类型 | 描述 |
| ------ | ---- | ------ | ---------------- |
| device_access_token | 是 | string | device_access_token ,获取方式见[获取设备调用凭证](#40436) |
| oper_id | 是 | string | 远程开门指令的操作id，用户发起远程开门操作后，企业微信给厂商的回调中会携带操作id。 |
| type | 是 | string |上报类型 remote_open_door 门禁app点击远程开门，scan_qrcode_open 扫码门禁开门 |
| errcode | 是 | uint32 | 远程开门事件的操作结果：门禁远程开门指令执行结果，0：成功， 1：失败 |
| errmsg | 否 | string | 远程开门事件的操作结果描述，如：设备离线等 |

**返回结果：** 

```
{
  "errcode": 0,
  "errmsg": "ok"
}
```

**参数说明：** 

| 参数名 | 描述 |
| ------------ | ----------- |
| errcode | 错误码 |
| errmsg | 错误码描述 |

>注意
>此接口只用于上报远程开门结果，不会自动产生门禁通行记录，开门成功后，服务商调用此接口上报远程开门结果后，还需同步调用[上报门禁通行记录接口](40277)，如果远程开门失败则不需要再调用上报门禁通行记录接口
