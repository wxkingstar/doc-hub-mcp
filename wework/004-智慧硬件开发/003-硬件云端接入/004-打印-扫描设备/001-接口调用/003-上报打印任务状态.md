上报打印任务的状态，企业微信会透传展示给用户。

**请求方式：**POST（**HTTPS**）
**请求地址：** https://qyapi.weixin.qq.com/cgi-bin/openhw/device/report_printer_job_status?device_access_token=DEVICE_ACCESS_TOKEN

**请求包体：**
```
{
    "jobid": "JOBID",
    "status": 1,
    "errcode": 1,
    "errmsg": "打印机缺墨"
}
```

**参数说明：**
 
| 参数名 | 是否必须 | 类型 | 描述 |
| ---------------- | ---- | ---------- | ---------------------------------------- |
| device_access_token | 是 | string | device_access_token ,获取方式见[获取设备调用凭证](#40436) |
| jobid| 是| String | 下载任务的id|
| status| 是| Uint32 | 任务状态：<br/>1：打印失败<br/>2：打印成功|
| errcode | 否| Uint32 | 错误码|
| errmsg | 否| String | 错误描述，必须是utf8编码|

>status为1时，必须要有errcode和errmsg，具体值可以自定义
status为2时，可以没有errcode和errmsg，如果有传值企业微信也会进行记录，建议成功时errcode为0。

**返回结果：**
```
{
   "errcode": 0,
   "errmsg": "ok"
}
```
**参数说明：**
 
| 参数名 | 类型 | 描述 |
| ---------------- | ---- | ---------------------------------------- |
| req_id | String | 透传请求的req_id |
| errcode | Uint32 | 错误码|
| errmsg | String | 错误描述|
