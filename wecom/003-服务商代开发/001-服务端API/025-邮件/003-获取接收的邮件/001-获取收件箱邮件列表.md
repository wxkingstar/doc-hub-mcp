由于用户可向应用所绑定的邮箱回复邮件，故支持应用获取**应用收件箱**的邮件列表。
该接口用于分页获取收件箱下，指定时间范围内的邮件id列表。

**请求方式：** POST（**HTTPS**）
**请求地址：** https://qyapi.weixin.qq.com/cgi-bin/exmail/app/get_mail_list?access_token=ACCESS_TOKEN

**请求包体**
```json
{
 "begin_time": 1668441600,
 "end_time": 1668527999,
 "cursor": "CURSOR",
 "limit": 100
}
```

**参数说明**

| 参数 | 必须 | 类型 | 说明 |
| ------------ | ------------ | ------------ | ------------ |
| access_token | 是 | string | 调用接口凭证 |
| begin_time | 是 | uint32 |开始时间，unix时间戳 |
| end_time | 是 | uint32 |结束时间，unix时间戳 |
| cursor | 否 | string |上一次调用时返回的next_cursor，第一次拉取可以不填 |
| limit | 否 | uint32 | 期望请求的数据量，默认值为100，最大值为1000 |

**权限说明**
- 自建应用需配置到“[可调用应用](#37468)”列表中的应用secret所获取的accesstoken来调用（[accesstoken如何获取？](#10013/第三步：获取access_token)）
- 第三方应用需具有“邮件”权限
- 代开发自建应用需具有“邮件”权限

**返回示例**

```json
{
 "errcode": 0,
 "errmsg": "ok",
 "next_cursor": "NEXT_CURSOR",
 "has_more": 0,
 "mail_list": [
 {
 "mail_id": "MAIL_ID"
 }
 ]
}
```
**参数说明**

| 参数 | 类型| 说明 |
| ------------ | ------------ |------------ |
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| next_cursor | string | 应用邮箱账号中邮件未读数 |
| has_more | uint32 | 是否还有更多数据。0-没有 1-有 |
| mail_list | obj[] | 邮件列表 |
| mail_list[].mail_id | string | 邮件id |
