启用了 “截屏/录屏管理”的企业可以通过本接口查询截屏操作记录。

**请求方式**：POST（**HTTPS**）
**请求地址**: https://qyapi.weixin.qq.com/cgi-bin/security/get_screen_oper_record?access_token=ACCESS_TOKEN

**请求包体**
```json
{
  "start_time": 166666666,
  "end_time": 166666667,
  "userid_list": ["zhangsan", "lisi"],
  "department_id_list" : [2, 3],
  "screen_shot_type": 1,
  "cursor":"ngLgjieajgieo",
  "limit":100
}
```

**参数说明**

| 参数 | 类型 | 是否必须 | 说明 |
| --------------------------------- | ------ | -------- | ----------------------------------------------------------------------------------------------------------- |
|access_token|string|是|应用调用凭证|
|start_time|int|是|开始时间|
|end_time|int|是|结束时间，开始时间到结束时间的范围不能超过14天|
|userid_list|array|否|需要查询的截屏操作者的userid，单次最多可以传`100`个用户。设置的userid需要在应用的可见范围内|
|department_id_list|array|否|需要查询的截屏操作者部门的department_id，单次最多可以传`100`个部门id。设置的department_id需要在应用的可见范围内|
|screen_shot_type|int|否|截屏内容的类型，不设置默认为全部</br>1: 聊天</br>2: 通讯录</br>3: 邮件</br>4: 文件</br>5: 日程</br>6: 其他|
|cursor|string|否|由企业微信后台返回，第一次调用可不填|
|limit|int|否|限制返回的条数，最多设置为`1000`|

**调用说明**
- 调用的应用需要满足如下的权限：

| 应用类型 | 权限要求 |
| -------- | ---- | 
|自建应用|配置到「!!#ff0000 截屏/录屏管理 - 可调用接口的应用!!」中 |
|代开发应用|暂不支持| 
|第三方应用|暂不支持|
- `返回结果中应用可见范围外用户相关的数据会被过滤掉，不会返回`

**返回示例**

```json
{
  "errcode": 0,
  "errmsg": "ok",
  "has_more": true,
  "next_cursor": "gejMjgLjgeigoejg",
  "record_list": [{
    "time": 16666666666,
    "userid": "zhangsan",
    "department_id" : 2,
    "screen_shot_type" : 1,
    "screen_shot_content" : "聊天",
    "system" : "iPad"
  }, {
    "time": 16666666666,
    "userid": "lisi",
    "department_id" : 3,
    "screen_shot_type" : 4,
    "screen_shot_content" : "文件（防泄露文件：test.pdf）",
    "system" : "iOS"
  }]
}
```
**参数说明**

| 参数 | 类型 | 说明 |
| ------- | ------ | ---------- |
| errcode | int32 | 错误码 |
| errmsg | string | 错误码说明 |
|has_more|bool|是否还有更多数据|
|next_cursor|string|仅has_more值为true时返回该字段，下一次调用将该值填到cursor字段，以实现分页查询|
|record_list.time|int|操作时间|
|record_list.userid|string|企业用户账号id|
|record_list.department_id|int|企业用户部门id。仅当department_id在应用可见范围内才返回|
|record_list.screen_shot_type|int|截屏内容的类型</br>1: 聊天</br>2: 通讯录</br>3: 邮件</br>4: 文件</br>5: 日程</br>6: 其他|
|record_list.screen_shot_content|string|截屏内容|
|record_list.system|string|企业用户的操作系统，目前支持返回的类型：iOS、iPad|
